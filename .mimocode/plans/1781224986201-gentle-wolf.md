# Project Structure Overview: Zotu

## Project Type
- **Language**: Rust (edition 2024)
- **Framework**: GPUI (GPU-accelerated UI framework from Zed editor)
- **Purpose**: Desktop music player application

## Directory Structure
```
D:\code\zotu/
├── src/                    # Main source code
│   ├── components/         # UI components (sidebar, titlebar, songview, now_playing, playbar, setting)
│   ├── play/               # Audio playback engine (player.rs)
│   ├── db/                 # SQLite database layer for metadata storage
│   ├── ui/                 # UI-related modules
│   ├── app.rs              # Main application logic
│   ├── config.rs           # Configuration handling
│   ├── theme.rs            # UI theming
│   ├── error.rs            # Error handling
│   ├── util.rs             # Utility functions
│   └── assets.rs           # Asset management
├── assets/                 # Application assets (icons, images)
├── .mimocode/              # MiMo Code agent configuration
├── Cargo.toml              # Rust project manifest
├── config.json             # Runtime configuration (music directory, volume, loop mode)
└── metadata.db             # SQLite database for song metadata
```

## Key Components
1. **UI Layer**: Built with GPUI framework, includes sidebar navigation, song view, now-playing display, and playbar
2. **Audio Playback**: Uses Rodio library with Symphonia decoder support
3. **Data Storage**: SQLite database (rusqlite) for storing song metadata
4. **Metadata Reading**: Uses lofty library to read audio file tags and album art
5. **File System**: Walkdir for scanning music directories

## Dependencies (from Cargo.toml)
- gpui: GPU-based UI framework
- rodio: Audio playback
- rusqlite: SQLite database (bundled)
- lofty: Audio metadata/tag reading
- image: Album cover image handling
- rfd: Native file dialogs
- serde/serde_json: Configuration serialization
- walkdir: Directory traversal for scanning music files

## Notes
- Comments in source code are in Chinese, suggesting Chinese-speaking developer
